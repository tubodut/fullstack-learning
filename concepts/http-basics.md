# HTTP基礎

## HTTPとは
WebサーバーとWebブラウザ間で、Webの情報をやり取りするためのプロトコル(通信規則)
動作は常に、クライアントが要求→サーバーが応答を返す
HTTPはTCP/IPでクライアントが送信するリクエストとサーバーが応答するレスポンスで通信が確立している。
リクエストにはヘッダーやメッセージの各部が存在し、行いたい操作をリクエストのメソッドで指定する。
レスポンスにはステータス行、ヘッダーフィールド、メッセージが存在し処理結果はステータスコードで表示される。

## HTTPメソッド
HTTPメソッドとはWebグラウザとサーバがやり取りをする時の通信手順である

GET：指定したターゲットをサーバーから取り出す
HEAD：指定したターゲットに関連するヘッダー情報を取り出す
POST：指定したターゲット(プログラム)にデータを送る
PUT：サーバー内のファイルを書き込む
DELETE：サーバーないのファイルを削除する
CONNECT：プロキシサーバ経由で通信を行う

### GET
- 用途: データを取得する
- 例: ユーザー一覧を見る、商品詳細ページを開く
- GASでの例: （太紀さんのシフト管理システムで考えると？）
空いているシフトの枠を取得する

### POST
- 用途: 新しいデータを作成する
- 例: 新規ユーザー登録、投稿を作成
- GASでの例: （太紀さんのシフト管理システムで考えると？）
GETで取得した情報をもとにGoogleフォームを作成する
確定したシフトを元に各ユーザーに対してシフト確定メールを作成する

### PUT
- 用途: 既存データを更新する
- 例: プロフィール編集、記事の更新
- GASでの例: （太紀さんのシフト管理システムで考えると？）
Googleフォームで作成した内容について、変更が出た際に更新を行う

### DELETE
- 用途: データを削除する
- 例: アカウント削除、投稿削除
- GASでの例: （太紀さんのシフト管理システムで考えると？）
Deleteの機構は作成していなかったです。

## HTTPステータスコード
Webサーバーがクライアントからのリクエストに対して、処理が成功したかエラーが発生したか追加の処理が必要なのかなど、結果や状態を伝える3桁の数字
これらのコードは100番台から500番台まであり、それぞれ意味が分かれている。
Webサイトの運用やトラブルシュートに不可欠なものとなっている

### 2xx: 成功
- 200 OK: クライアントからのリクエストがサーバーで正常に処理され、要求された情報がレスポンスとして返された状態を示す。
- 201 Created: クライアントのリクエスト(主にPOST)が成功し、サーバー上で新しいリソースが正常に作成されたことを示す。

### 4xx: クライアントのエラー
- 400 Bad Request: クライアントからサーバーへ送られたリクエストの構文や内容に問題があり、サーバーがそのリクエストを処理できなかった場合に返される。
- 404 Not Found: サーバーがリクエストされたページやファイルを見つけられないことを示すエラーコード

### 5xx: サーバーのエラー
- 500 Internal Server Error: サーバー側で予期せぬ問題が発生し、クライアントからのリクエストを正常に処理できなかったことを示すエラー
